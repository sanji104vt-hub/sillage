# Sillage 香水商品データ編集基準

## 目的

商品詳細データを、出典の強さと確認日を明示しながら再現可能な方法で補完する。確認できない情報は推測せず、欠損のまま保持する。

## 情報源の優先順位

1. 日本国内のブランド公式商品ページ
2. ブランド公式の国内プレスリリース
3. 海外ブランド公式商品ページ
4. 日本国内の正規代理店
5. 百貨店・正規取扱店
6. 信頼できる大手小売店

非公式ブログ、口コミまとめ、生成AI記事、マーケットプレイス出品者の商品説明は、商品事実の主出典にしない。楽天などのマーケットプレイスは購入リンクの同一商品確認に限って使用し、濃度・容量・ノート・発売年の主出典にはしない。

## 国内公式ページがない場合

海外公式ページから香水濃度、容量、ノート、商品説明を確認してよい。ただし、海外ページの価格や在庫を日本国内の価格・販売状況として扱わない。

国内価格や国内販売状況を確認できない場合は、商品ページで次の趣旨を表示する。

> 海外公式情報を参照しています。日本国内の価格・販売状況は掲載していません。

海外公式ページの国・言語による自動転送やBot対策による403は、URLのドメイン、パス、商品名が一致していれば直ちにリンク切れとは判定しない。

## 国内参考価格

- 国内参考価格は必須ではない。
- 日本国内のブランド公式、正規代理店、百貨店・正規取扱店で容量と税込価格を確認できる場合だけ記録する。
- 価格は容量ごとの `referencePriceYen`、確認日は `priceVerifiedAt`、根拠URLは `sourceUrl` に保存する。
- 確認できない場合は容量だけを表示する。
- 海外価格の円換算、マーケットプレイス価格、セール価格を国内参考価格として掲載しない。
- 既存の概算文字列 `price` は、今回の確認済み価格として扱わない。

## 事実と編集判断

### 事実として保存できるもの

- 公式に記載された香水濃度、容量、ノート、商品説明、発売年
- 国内の優先情報源で確認した税込価格
- 既存データと公式情報が明確に矛盾する場合の訂正

### 編集判断に使用できる根拠

- 公式ノート構成
- 公式の商品説明
- 既存の香調
- 既存のシーン
- 既存の季節
- 香水濃度

`recommendedFor` と `notRecommendedFor` は各項目に `basis: "editorial"` を付ける。UIでは「Sillage の見立て」と表示し、公式情報と区別する。

### 編集判断に使用しないもの

- 根拠のない年齢指定や性別の断定
- 「モテる」など結果の断定
- 芸能人の使用情報
- 医学的・心理的効果
- 実測していない持続時間や拡散力
- 肌質による断定

## おすすめする人

- 最大3件。
- 香調の好み、使用場面、香りの印象、季節・時間帯のいずれかを、公式情報と既存属性に結び付けて表現する。
- 商品の優劣や利用者の属性ではなく、香りとの相性を説明する。

例：「落ち着いたウッディ系を日常使いしたい人」

## おすすめしにくい人

- 最大3件。
- 商品を否定せず、求める香調や印象との不一致として表現する。
- 「初心者には向かない」のような利用者評価は避ける。

例：「軽く爽やかな香りだけを求める人には、重く感じられる可能性があります」

## 注意点

- `cautions` には商品固有の確認事項だけを最大3件保存する。
- 海外公式情報だけを参照した場合の国内価格・販売状況、リフィル対象、限定容量、公式ページの販売終了・在庫表示などを記録できる。
- 「香り方には個人差がある」「価格は変更される」「肌に異常がある場合は使用しない」といった全商品共通事項は保存しない。共通UIまたはサイトポリシーで扱う。
- 「不明」「未確認」など、欠損値のプレースホルダーに見える表現は使わない。「掲載していません」「確認対象に含めていません」のように掲載範囲を説明する。

## profile

評価方法を確定するまで、全軸を `null` としUIへ表示しない。

```json
{
  "lightToRich": null,
  "freshToSweet": null,
  "subtleToBold": null,
  "dailyToDistinctive": null,
  "youthfulToMature": null,
  "method": "editorial-v1"
}
```

商品説明やノートから数値を自動生成しない。

## 購入リンク

### 公式

- 商品詳細ページへ直接到達するURLを優先する。
- ドメイン、URLパス、ページの商品名・濃度を確認する。
- 検索結果より商品詳細ページを優先する。
- `type: "product"` と `verifiedAt` を記録する。

### 楽天

- ブランド、商品名、濃度を照合する。容量違いは同一商品であれば許容する。
- 別商品や別濃度へ到達するURLは使わない。
- 既存アフィリエイトパラメータを維持する。
- 商品詳細へのリンクは `type: "product"`、商品を特定できず検索結果を使う場合は `type: "search"` とする。
- 表示文言は種別と一致させる。productは「楽天市場で商品を見る」、searchは「楽天市場で検索」とする。

## 情報源

各情報源には `url`、`publisher`、`title`、`accessedAt`、`supports` を記録する。正規代理店や百貨店を使う場合は `sourceType` も記録し、ブランド公式と区別する。

同一商品内で同じURLを重複登録しない。複数項目を支える場合は1件の `supports` にまとめる。

## 更新日

- `updatedAt`: 商品データを実際に変更した日
- `verifiedAt`: 商品の情報源と購入リンクを確認した日
- `priceVerifiedAt`: 国内参考価格を確認した日
- `sources[].accessedAt`: 各情報源へアクセスした日

日付はISO 8601の `YYYY-MM-DD` で保存する。全商品への一括固定日は使用せず、実際に変更・確認した商品だけを更新する。

## リンク監査の判定

- `ok`: 200応答で目的ページへ到達
- `redirect`: リダイレクト後に有効なページへ到達
- `blocked`: 403。Bot対策の可能性があり手動確認が必要
- `rate_limited`: 429。時間を空けて再確認
- `not_found`: 404または410
- `timeout`: 設定時間内に応答なし
- `manual_review`: 商品同一性、公式ドメイン、遷移先を自動判定できない

403と429をリンク切れと断定しない。外部サイトへ連続アクセスせず、低い同時実行数、待機時間、限定した再試行回数を使う。

## 変更範囲の確認

- 補完対象の商品IDを先に固定する。
- 商品配列の並び順、商品ID、slugを変更しない。
- 対象外商品へ補完フィールドを追加しない。
- 生成前後に対象商品IDの差分を照合する。
- GA4、Search Console認証タグ、canonical、構造化データを維持する。
